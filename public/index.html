Reusable Web Components: How to Store and Manage Common Website Elements in a GitHub Repository

Posted on 11 April 2023 by LakiDIV

To achieve this, you can create a new GitHub repository to store all of your common website components such as navigation bars and footers.
First, create a new repository on GitHub and give it a name that reflects its purpose, for example, "website-components". Next, create a new directory inside the repository for each component that you want to store. For example, you can create a directory called "nav-bar" for your navigation bar components, and another directory called "footer" for your footer components. Inside each directory, you can add the different variations of the component. For example, inside the "nav-bar" directory, you can create subdirectories for different types of navigation bars, such as "top-nav" or "side-nav", and then add the HTML, CSS, and JavaScript files for each variation. Once you have added all of your components, you can use the repository as a submodule in your other projects. To do this, you can add a submodule reference to your other repositories using the following command:

git submodule add
This will create a reference to the "website-components" repository inside your other projects. You can then use the components from the repository in your projects by referencing the files in the submodule. Finally, whenever you make changes to your common components in the "website-components" repository, you can update the submodule reference in your other projects using the following command:
git submodule update --remote
This will fetch the latest changes from the "website-components" repository and update the submodule reference in your other projects.

With this approach, you can easily manage and reuse common components across multiple projects and keep them in sync with updates made to the components in the "website-components" repository.
